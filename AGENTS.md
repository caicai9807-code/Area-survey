# Area Survey 仓库协作规则

本仓库是供热面积普查产品原型。后续 Codex 会话必须先理解业务与当前实现，再按用户逐页指令维护；不得把本仓库当作一次性页面生成任务。

## 1. 修改前必读

按顺序完整阅读：

1. `AGENTS.md`
2. `docs/面积普查项目上下文.md`
3. `docs/计划外面积普查整体流程.md`
4. `docs/计划外面积普查状态机.md`
5. `docs/计划外面积普查页面清单.md`
6. `docs/计划外面积普查数据模型.md`
7. `docs/原型设计与代码规范.md`
8. `docs/待确认问题清单.md`
9. 与本次页面直接相关的既有功能规格、系统建设方案及当前源码

同时执行 `git status --short --branch` 和 `git diff --check`。当前工作区可能包含用户尚未提交的成果，严禁覆盖、回退或顺手整理无关改动。

## 2. 需求与授权门禁

- 业务需求、功能想法或业务修改不得直接编码。
- 第一步“需求澄清”：从业务场景、用户痛点、异常流程、权限/性能约束等维度提出 3–5 个关键问题。
- 第二步“方案输出”：用户回答后输出《功能规格说明书》和《系统建设方案》。规格必须含功能定义、用户故事和 Mermaid 业务流程图；建设方案必须含产品架构、模块划分、核心交互逻辑和 Ant Design 组件选型。
- 第三步“授权确认”：明确询问“方案是否通过？”，只有用户确认通过后才能实施。
- 纯样式调整不需要上述确认，但仍须先检查当前文件和影响范围。

## 3. 实施边界

- 每次只实施用户明确指定的页面、功能及其不可缺少的公共能力。
- 不得提前生成下一页面，不得依据自己的理解扩展未确认业务。
- 如新指令与既有文档、代码或已确认规则冲突，先写入 `docs/待确认问题清单.md` 并向用户确认，不得静默选边。
- 以当前工作区最新文件为实现事实，以用户最新明确口径为目标规则；必须显式区分“已实现”“目标态”“待确认”。
- 新页面或新逻辑必须复用现有视觉规范、公共样式、状态字典和数据模型，不得另起一套相似实现。
- 不引入外部网络依赖或大型框架；原型需保持 HTML、CSS、原生 JavaScript 可直接运行。

## 4. 单页任务开始前的实施理解

编码前向用户简要说明并等待确认，至少包含：

1. 页面所处业务阶段；
2. 使用角色和权限范围；
3. 页面入口；
4. 完成后的去向；
5. 读取和写入的数据；
6. 涉及状态；
7. 本次修改和新增文件。

若用户指令已经明确且已完成方案授权，可直接给出这份实施理解后实施；不得顺带扩大范围。

## 5. 业务底线

- 计划内与计划外任务来源必须可追溯；计划外主任务下的普查对象独立执行、独立审核。
- 计划外原因固定为：面积变化、一管到户、新开户及增容、趸售用户；一项任务只能选择一种原因。
- 临时对象不得写入正式站点档案；正式用户编码和卡号不由普查系统生成。
- 无正式编码或卡号不影响普查及审核，但不得同步最终面积；按“先普查、后建卡、补录后同步”处理。
- 同站点同年度允许多轮普查；历史轮次保留，只有最后一次审核通过的有效轮次可参与同步。
- 草稿或未下发数据才可删除；已完成数据不可删除，只能按授权作废并保留历史。
- 退回、撤回、作废、审核、同步等关键动作必须保留原因、操作者、时间和前后状态。

## 6. 设计与代码规范

- 视觉保持紧凑型 Ant Design 风格后台：顶部栏、220px 左侧导航、浅灰蓝背景、小圆角、高信息密度、表格横向滚动、固定操作列。
- 字段名与内容尽量同一行，表格内容不随意换行，不通过删字段解决宽度问题。
- 优先复用 `area-survey-detail.css`、`area-survey-role-tasks.css`、`area-survey-survey-fill.css` 等公共能力；页面特有样式使用独立前缀。
- 新增路由必须使用可离线工作的相对路径，并验证入口、返回、空数据和非法参数路径。
- 不新增平行的状态文案或 localStorage key；确需新增时先更新数据模型和状态机文档。

## 7. 修改后的必做事项

1. 更新 `docs/计划外面积普查页面清单.md` 中对应页面的状态、入口、出口和依赖。
2. 更新 `docs/面积普查项目上下文.md` 的“变更记录”；涉及模型或状态时同步更新对应文档。
3. 新增或解决待确认项时更新 `docs/待确认问题清单.md`，保留结论来源和日期。
4. 运行 `git diff --check`。
5. 对独立 JS 和所有改动页面的内联脚本执行语法解析；按风险补充浏览器交互验证。
6. 交付时说明实际修改文件、验证结果、尚未验证内容和仍待确认事项。
