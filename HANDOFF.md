## 2. 项目与任务目标

### 2.1 项目定位

本项目是“面积普查与检查管理系统”的多页面高保真原型，用于需求评审、业务闭环确认和研发沟通。

- 主入口：`area-survey-plan-list.html`
- 技术形态：静态 HTML + CSS + 原生 JavaScript
- 数据模拟：URL 查询参数、页面内数组、`localStorage`、`sessionStorage`
- 无构建工具、无真实后端、无真实鉴权、无真实文件服务
- 视觉：紧凑型 Ant Design 风格中后台，56px 顶栏、220px 侧栏、宽表横滚、固定操作列

### 2.2 总体业务目标

1. 建立正常年度普查的计划、站点任务、人员分配、填报、两级审核和结果沉淀闭环。
2. 建立计划外普查的独立主任务和执行入口，同时复用正常普查的公共执行与审核能力。
3. 支持自管站、用户站、对公用户及目标态趸售对象的差异化填报。
4. 支持同站点同年度多轮普查、当前有效轮次、先普查后建卡、专项审批和最终面积同步的目标业务。
5. 保持任务来源、原因、轮次、审核、退回、作废和同步全程可追溯。

## 9. 踩过的坑

### 9.1 旧文档口径可能已失效

旧口径曾允许任一原因混选三类站点，并把“燃气替代”作为原因。最新口径是四类标准原因并按原因限制对象和模板；必须通过标准编码和历史别名兼容，不能直接删除旧值。

历史映射：

```text
面积变化 → AREA_CHANGE
一管到户（自管站）/direct → DIRECT_TO_HOUSEHOLD
新开户/new → NEW_ACCOUNT_OR_CAPACITY
燃气替代/gas/GAS_REPLACEMENT → WHOLESALE_USER
```

### 9.2 临时对象不是正式站点

- 临时对象嵌入计划外主任务 `temporaryStations[]`。
- `stationIds[]` 可能同时含正式站点 ID 和临时 ID。
- 解析必须使用 `outPlanStation(id, task)`，不能只查全局站点数组。
- 后续建卡必须建立映射，不能把 `tmp-*` 原地改成正式 ID。

### 9.3 不要破坏 localStorage 兼容

主要 key：

```text
areaSurveyPlans
areaSurveyTasks
areaSurveyHiddenBasePlans
areaSurveyTaskStates
areaSurveyOutPlanTasks
areaSurveyOutPlanChildStates
areaSurveyDraft:{taskId}
areaSurveyUserDraft:{taskId}
areaSurveyCorporateDraft:{taskId}
areaSurveyOutPlanRole
outPlanNotice
```

确需新结构时先更新数据模型和状态机，增加 `schemaVersion` 与迁移逻辑，不能直接换 key。

### 9.4 来源链路容易串页

- 计划内与计划外执行列表必须隔离。
- 复用填报/详情页时必须携带明确 `source`。
- 返回地址要从真实来源计算，不要拿 `source=office` 代替计划外来源。
- 写 localStorage 后再导航，不要依赖返回页才落库。

### 9.5 三类对象不能简单共用所有 UI

- 自管站有门头图和平面图。
- 用户站有未在网统计和用户站核查表。
- 对公用户侧重签字盖章汇总表。
- 一管到户目标态虽然对象来源为用户站，但使用自管站模板。
- 趸售目标态是站下多项目，不等同于现有单个用户站填报页。

### 9.6 列表与样式

- 不通过删字段解决宽度问题；使用横向滚动。
- 操作列固定时，表头和内容需要不同 `z-index`，行 hover 背景要同步。
- 字段和值尽量同一行，表单维持 3–4 列紧凑布局。
- 状态必须有文字，不能只靠颜色。
- 序号只用渲染索引，禁止写入业务对象。

### 9.7 浏览器与本地预览

- `file://` 预览可能受浏览器安全策略限制，优先使用本地静态服务。
- 浏览器已有 `127.0.0.1:4173` 标签不代表服务仍在运行；连接失败时先确认服务进程。
- 临时服务验证完成后应停止，避免残留端口进程。
- 浏览器回归时不要破坏用户当前标签和 localStorage；如占用用户标签，结束前恢复原 URL。
